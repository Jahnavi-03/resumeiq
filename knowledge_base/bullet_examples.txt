RESUME BULLET POINT EXAMPLES — STRONG VS WEAK
==============================================

PURPOSE: These examples teach the difference between weak vague 
bullets and strong quantified bullets following XYZ format.
Use these as templates when rewriting candidate bullets.

SECTION 4: REWRITING RULES

When rewriting a weak bullet always:
1. Start with a strong action verb (Built, Designed, Led, 
   Optimized, Architected, Developed)
2. Add what specifically was built or done
3. Add the technology stack used
4. Add the scale (how much data, how many users, how large)
5. Add the measurable result (time saved, cost reduced, 
   accuracy improved, reliability achieved)
6. Keep to 2 lines maximum — recruiters skip long bullets

SECTION 5: ACTION VERB LIST BY CATEGORY

BUILDING: Built, Developed, Created, Designed, Architected,
          Implemented, Engineered, Constructed, Established

IMPROVING: Optimized, Reduced, Improved, Accelerated, 
           Streamlined, Enhanced, Upgraded, Modernized

LEADING: Led, Managed, Directed, Coordinated, Mentored,
         Guided, Supervised, Spearheaded, Championed

DELIVERING: Shipped, Launched, Deployed, Released, Delivered,
            Executed, Completed, Finalized

ANALYZING: Analyzed, Investigated, Identified, Evaluated,
           Assessed, Researched, Diagnosed, Discovered

SECTION 6: NUMBERS CHEAT SHEET
When you cannot remember exact numbers use these approaches:

Team size: "team of X engineers"
Data volume: "Xmillion/billion records"
Time improvement: "reducing from X to Y hours/minutes"
Cost savings: "saving approximately $X annually"
User scale: "serving X users/patients/customers"
Coverage: "across X systems/databases/tables"
Reliability: "achieving X% uptime/accuracy"
Speed: "X requests per second"
